**빌드(build)** 는 개발용 소스 코드를 브라우저나 서버가 실제로 실행할 수 있는 **산출물(artifact)** 로 변환하는 과정이고, **배포(deploy)** 는 그 산출물을 사용자가 접근할 수 있는 환경에 올려 제공하는 과정입니다.

## 빌드 (Build)

개발 단계의 코드는 그대로 배포하기에 비효율적이거나 브라우저가 이해하지 못합니다. 빌드는 이를 최적화된 실행 가능한 형태로 바꿉니다.

- **트랜스파일** — TypeScript·JSX·최신 문법을 브라우저 호환 JS로 변환(Babel, swc, tsc).
- **번들링** — 흩어진 모듈을 묶고 의존성을 해결.
- **최적화** — minify(공백·이름 축약), 트리 셰이킹, 코드 스플리팅, 이미지·CSS 처리.
- **결과물** — `dist/`·`build/` 폴더의 정적 파일(HTML·JS·CSS·에셋) 또는 서버 번들.

## 배포 (Deploy)

빌드 산출물을 실행·서비스 환경에 올리는 과정입니다.

- **정적 사이트** — 빌드된 파일을 CDN/정적 호스팅(Vercel, Netlify, S3 등)에 업로드.
- **서버 앱** — 서버에 번들을 올리고 프로세스를 실행(Node 서버, 컨테이너 등).
- **CI/CD** — 커밋을 트리거로 빌드·테스트·배포를 자동화(GitHub Actions 등). 무중단 배포·롤백·환경(dev/staging/prod) 분리가 핵심.

## 전형적 흐름

```bash
# 1. 의존성 설치
bun install
# 2. 빌드 (소스 → dist)
bun run build
# 3. 배포 (산출물을 호스팅/서버로)
# CI가 dist를 CDN/서버로 업로드 → 사용자에게 제공
```

빌드는 "**무엇을 내보낼지** 만드는 단계", 배포는 "**그것을 어디에 올려 제공할지** 의 단계"로 구분하면 명확합니다.
