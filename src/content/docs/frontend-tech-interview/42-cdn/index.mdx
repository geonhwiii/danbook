**CDN(Content Delivery Network)** 은 전 세계 여러 지역에 분산 배치된 **캐시 서버(엣지 서버) 네트워크**입니다. 정적 콘텐츠를 사용자와 가까운 서버에서 제공해 빠르게 전달합니다.

## 동작 원리

원본 서버(origin)의 콘텐츠를 각 지역의 엣지 서버에 복제해 두고, 사용자는 **물리적으로 가장 가까운 엣지 서버**에서 콘텐츠를 받습니다. 첫 요청 때 원본에서 가져와 캐싱하고, 이후 요청은 캐시에서 바로 응답합니다.

## 성능 개선 기여

- **지연 시간(latency) 감소** — 사용자와 가까운 서버에서 받으므로 네트워크 왕복 거리가 짧아집니다.
- **원본 서버 부하 분산** — 트래픽을 엣지가 흡수해 원본 서버 부담을 줄입니다.
- **대역폭 절감과 캐싱** — 자주 쓰는 JS·CSS·이미지·폰트 등 정적 자산을 캐시해 반복 전송 비용을 줄입니다.
- **트래픽 급증 대응** — 분산 구조라 갑작스러운 트래픽에도 안정적입니다.
- **보안** — 많은 CDN이 DDoS 완화, TLS 종료 등 부가 기능을 제공합니다.

## 프런트엔드에서 주로 캐싱하는 것

- JS·CSS 번들, 이미지·폰트 등 **정적 자산**
- 빌드된 정적 HTML(SSG 결과)

자주 바뀌지 않는 정적 자산일수록 CDN 효과가 큽니다. 파일명에 해시를 붙이는 **캐시 버스팅(cache busting)** 으로 갱신 시점을 제어합니다.
